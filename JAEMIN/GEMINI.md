1. 목적
- Oracle DB 에서 NVH Vehicle Interior Noise 실차평가와 타이어 단품 실내시험 평가 결과와 타이어의 정보 (Spec_no, 타이어 구조 정보, 재료 등), 시험조건 등 데이터를 불러와서 타이어 개발 차수별로 성능과 History를 보기좋게 보여주는 홈페이지를 제작.
- 원래는 Oracle DB에서 불러와야 하지만 지금은 아직 DB 제작이 되지않아서 임의의 값으로 제작.
- 이 홈페이지는 Python Flask 서버와 HTML, CSS, Javascript로 구성되어야 함.
2. 결과물
- 왼쪽에는 트리구조의 형식으로 차량 코드명 (CN8, NX5, SX3), 규격(17인치, 18인치, 19인치), 개발 차수 (1s, 2s, 3s) & 평가날짜(랜덤), 그리고 각 차수 하위에 해당 Spec 리스트 순서로 표시하고 트리는 기본적으로 펼쳐져 있음. 각 트리마다 접고 펼 수 있는 +/- 버튼 필요. 규격, 개발 차수, 그리고 개별 Spec 오른쪽에 체크박스를 구현하여 사용자가 보고 싶은 데이터만 필터링할 수 있게 해줘. 규격 체크박스가 해제된 상태여도 개발 차수나 개별 Spec을 체크하면 해당 상위 규격이 자동으로 선택(다른 규격은 해제)되어 데이터가 즉시 보여야 함. 특정 Spec만 체크 해제하면 해당 데이터가 표와 Plot에서 즉시 사라져야 함.
- 맨 위쪽에는 제목을 큰 글자, Bold 체 사용하여 'Omnia' 표시.
- 제목 및 트리를 제외한 나머지 공간의 위쪽에는 맨 아래 Scatter와 Comment를 날짜 별로 볼 수 있는 게시판을 만들어줘. 게시판에 저장된게 보일 때는 Scatter와 작성자, 날짜만 보여주고 해당 게시글은 각 차량 및 규격별로 엄격하게 필터링되어야 해. 게시판 레이아웃은 사이드바에서 '체크된 차수' (1s, 2s, 3s...) 섹션만 세로로 나열되게 하며, 만약 여러 차수를 선택한 뷰에서 저장한 경우 맨 상단에 'Multi-Round Comparison' 섹션을 만들어 따로 관리해줘. 각 그룹 섹션 내부에서는 게시글이 가로(오른쪽)로 나열되게 해줘. 게시판 전체 영역은 고정된 높이를 가지며 차수가 많아질 경우 세로 스크롤이 생겨야 하고, 각 차수 그룹 내에서 게시글이 많아질 경우 가로 스크롤이 생겨야 함. 또한 화면 배율 100% 기준으로 게시판 영역은 기존 크기의 약 절반 정도로 콤팩트하게 구성해줘. 그리고 해당 게시글에 마우스를 올리면 뒤집히면서 Comment가 보이도록 해주고, 해당 게시글을 클릭하면 해당하는 차량, 규격 및 **저장 당시 선택했던 모든 개발 차수**가 정확하게(다른 규격의 차수는 켜지지 않게) 자동으로 활성화되어 데이터 창으로 전환되어야 하며, 저장된 Scatter 이미지는 **세로 크기를 1000px로 대폭 키워 단 하나의 'Restored Insight' 박스**로 즉시 나타나도록 해줘(새로운 게시글 클릭 시 이전 내용은 자동으로 사라짐).  
- Plot 아래에는 차수 및 스펙별로 변경된 구조들만 표로 정리해줘. 이번에는 랜덤이니까 C/C 구조, Belt 구조, Belt Angle, B/F Height, 보강벨트 재질, 보강벨트 구조 만 표에 넣어서 랜덤으로 넣어줘. 그리고 바로 왼쪽 스펙에서 변경된 사항은 연한 색으로 표시되게 해줘.
- 구조표 아래의 왼쪽에는 개발차수 및 스펙에 따른NVH Vehicle Interior Noise 실차평가 결과를 표로 보여줘. 결과는 Booming, Cavity Peak, Cavity , Rumble, Pattern Noise 임. Cavity Peak를 제외한 항목들은 모두 RMS 값임. 개발차수와 스펙에 따른 성능 결과값은 50~70 사이의 값으로 랜덤으로 줘. 소숫점 첫째 자리까지만 표기.
- 구조표 아래의 오른쪽에는 개발차수 및 스펙에 따른 단품 성능 결과표를 보여줘.성능결과는 ABN, SBN, Modal, Cleat Impact, F/T 만 표로 표시해줘. 모든 값은 랜덤으로 넣자.
- 맨 아래에는 사용자가 원하는 평가 결과를 Scatter에 뿌려서 볼 수 있게 하는 기능을 넣어줘. X축과 Y축에 들어갈걸 선택할 수 있고, Plot도 원하는 만큼 생성하고 지울 수 있게 해줘. 각 Scatter마다 comment 를 작성할 수 있게 하고 각 Scatter 와 comment를 사용자가 설정 및 작성하면 Save를 할 수 있게해줘. Save된 내용은 DB를 만들어서 관리할 수 있게 해줘. 데이터가 계속 축적되어서 다른 사용자들도 볼 수 있게 해야함.
