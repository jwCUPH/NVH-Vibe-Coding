개별 EXCEL 파일을 집어넣으면, plain html 파일을 제작해주는 웹페이지를 만든다.

웹페이지는 python flask 서버를 통해 구동되며, 되도록이면 가볍게 구동되는 동적 웹페이지를 구성한다.
이때 html, css, script 를 통해 동작하는 웹페이지가 되고, 각 파일은 수정이 용이하게 모듈(?)형식으로 제작

웹페이지를 통해 만들어진 html 파일은 DB에 저장하여, 서버에서 언제든 꺼내 볼 수 있도록 만들 것임!

코드의 구성을 마치면, 코드를 실행 할 수 있는 방법을 구체적으로 설명해줘

단계별로 구현을 해보자.
각 구현이 끝나면, 다음 구현을 하기전 단계별 확인 절차를 거쳐라

1단계 : 웹페이지 구현
 - 웹페이지에는 파일을 업로드 할 수 있는 칸을 둔다.
 - 파일은 Day1, Day2, Day3 등 파일을 여러개 업로드 할 수 있게 한다.
 - 이때 Day1 칸만 구성하고, Day2 부터는 Day1 칸 옆에 + 및 - 버튼을 두어 추가/삭제 할 수 잇다.
 - 각 행이 늘어나면 늘어난 칸에 + 및 - 버튼을 두어 추가 행을 구성하게 한다. Day1 칸 옆에는 + 만 두어 삭제 불가 하게 한다.
 - 각 파일의 업로드 칸은 각 행으로 구성한다
 - 파일의 업로드 후에 파일을 읽을 수 있는지 업는지 확인하여 각 행의 우측편에 초록색 'OK' 혹은 붉은색 'NG' 표기를 띄워준다.
 - 각 Day 별로 엑셀파일이 업로드 되었다면. 하단에 해당 행과 동일한 너비의 'Excel - VPR 추출' 이라는 버튼을 둔다
 - 위 버튼을 누르면 각 Excel 파일의 파싱을 진행한다.
 - 파싱하여 만들어지 html는 엑셀에서 파싱이 불가하여 사용자가 수동으로 기입되야 하는 항목이 존재한다.
 - 각 항목들을 사용자가 수동으로 기입하고 나면, 최종 html 파일로 변환하는 버튼을 둔다 (저장기능 수행)
 - 최종 저장된 파일은 서버 업로드 버튼을 두어 업로드 하고, 수정을 하고 싶다면 서버에 저장된 html 파일 중 선택하여 수정을 진행한다.

2단계 : 엑셀파일 파싱 구현
 - 엑셀파일을 파싱하면 최종적으로 각 카테고리별 정보가 기입된 하나의 리포트 형식의 html 파일이 제작된다.
 - plain html은 게시글 형태의 텍스트와 이미지로 구성된 페이지 이다.게시글의 카테고리는 'Example_Plain.txt' 파일을 참고한다
 - 'Example_Plain.txt' 파일에는 각각 작성해야할 카테고리별로 Excel 파일에서 어떤 sheet 의 어떤 행/열 을 가져와야 하는지가 '{ }' 안에 작성 되어 있음
 - { } 안에 작성되는 표기 룰은 'Sheet name'/'해당행열정보' 로 기입된다 (Ex, {DB/A3})
 - 파일에서 파싱을 하지 못하고, 직접 입력해야 할 경우에는 '{사용자 입력}' 이라고 표기 되어 있다.
 - 시트명과 해당 행열 정보에서 계산이 필요하거나, 범위가 지정되어 새롭게 구성이 필요한 경우는 '해당행열정보' 에 필요한 계산 로직및 구성방법에 대해 기입되어 있음
 - 카테고리중 Day1, Day2 로 표기된 항목은 업로드된 엑셀파일의 Day1, Day2 에서 각각 파싱해 오면됨. 그리고 파일이 하나만 업로드 된경우에는 일자 구분 없이 카테고리를 나누지 않고 표기
