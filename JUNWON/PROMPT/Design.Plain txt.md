Plain html 의 디자인 가이드임

- 작성되는 글자의 폰트 및 글자크기 및 줄간격은 같다
- 작성되는 보고서 형태의 html 은 구동되는 웹페이지에 종이 형태의 문서가 존재하는 식으로 구성
- 종이에는 회색 외곽선과 그림자가 존재하여 튀어나오느 느낌
- 창 너비에 맞춰 보고서의 크기가 유동적으로 조절되도록 구성 (Responsive Width)
- 레이아웃을 넘어가는 경우 상하, 좌우 드래그 가능하게 구성
- 해당 보고서 형태에 표기될 정보는 'Example_Plain txt.txt' 파일의 형식을 엄격히 준수한다.
- **편집 편의성**: '평가장소/노면', '추가분석', 'Test summary' 등의 편집 버튼은 보고서 레이아웃 외부에 배치하여 최종 결과물에 영향을 주지 않도록 한다.
- **자동 서식**: 로드노이즈 및 패턴노이즈 입력 칸은 줄바꿈 시 자동으로 '- ' 기호가 생성되도록 구현한다.
- 불필요한 가이드 문구(예: 자동 추출, 기본 정보 등)를 배제하고 텍스트와 들여쓰기 형태를 동일하게 유지한다.
- '최종 HTML 파일로 변환' 버튼은 종이 레이아웃 바깥 하단에 하나만 배치한다.
